# Task Completion Checklist

Before considering a task finished, ensure the following:

1.  **Functionality:** Verify that the changes work as intended.
2.  **Type Checking:** Run `npm run build` (or `tsc`) in both `backend` and `frontend` to ensure no type errors.
3.  **Linting:** Run `npm run lint` in both `backend` and `frontend` and fix any warnings or errors.
4.  **Formatting:** Ensure code follows Prettier rules.
5.  **Documentation:** Update README or other documentation if necessary (in Japanese).
6.  **Git:** Check `git status` and prepare a clear commit message explaining the "why".
7.  **Clean up:** Remove any temporary debug logs or files created during development.
